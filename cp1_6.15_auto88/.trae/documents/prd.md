## 1. 产品概述

手写甘特图工具——将手写待办事项清单自动识别并转换为可交互的甘特图，解决项目管理中手写计划与数字化跟踪之间脱节、人工录入耗时且易出错的问题。
- 目标用户：项目经理、团队负责人、个人效率工作者
- 核心价值：从手写到甘特图1.5秒内完成转换，消除手动录入瓶颈

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 使用全部功能 |

### 2.2 功能模块
1. **主页面**: 白板手写区 + 甘特图面板 + 左侧分组面板 + 顶部搜索筛选 + 右上角导出按钮

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 白板面板 | A4虚拟白板（210x297mm，背景#f7f3e3），支持手写输入，停止2秒后自动OCR识别，解析任务名称/起始日期/截止日期/进度百分比 |
| 主页面 | 甘特图面板 | 自动生成甘特图条块，按优先级分配颜色（高#e74c3c/中#f39c12/低#2ecc71），支持拖拽调整时间、双击编辑、进度渐变填充，时间轴按周显示 |
| 主页面 | 分组面板 | 左侧面板（220px，背景#fafafa），创建项目分组，支持展开/折叠，三角形箭头旋转动画0.2s |
| 主页面 | 搜索与筛选 | 顶部搜索框（圆角6px，占位符#aaa）和进度范围滑块（轨道150px） |
| 主页面 | 导出功能 | 右上角圆形导出按钮（直径36px，悬停#e8e8e8），支持PNG（2x分辨率）和PDF导出，导出时显示"导出中…"动画 |

## 3. 核心流程

用户在白板上手写待办事项 → 停止操作2秒触发OCR → 后端Tesseract.js识别文本 → 前端解析字段生成任务 → 甘特图自动渲染条块 → 用户可拖拽/双击编辑 → 按组筛选查看 → 导出PNG/PDF

```mermaid
    "A[用户在白板手写待办事项]" --> "B[停止操作2秒]"
    "B" --> "C[自动截图发送至OCR服务]"
    "C" --> "D[Express后端调用Tesseract.js识别]"
    "D" --> "E[解析文本提取任务字段]"
    "E" --> "F[生成任务数据写入Zustand Store]"
    "F" --> "G[甘特图自动渲染条块]"
    "G" --> "H{用户交互}"
    "H" --> "I[拖拽调整时间]"
    "H" --> "J[双击编辑任务]"
    "H" --> "K[分组筛选查看]"
    "I" --> "G"
    "J" --> "G"
    "K" --> "L[导出PNG/PDF]"
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：暖白（#f7f3e3）、木质棕（#c8a882）、深灰（#2d2d2d）
- 按钮样式：圆角6px，轻微投影（0 2px 6px rgba(0,0,0,0.1)）
- 字体：手账风格，使用衬线体搭配无衬线体
- 布局：两栏布局，可拖拽分割线调整比例
- 整体氛围：温暖手账风，纸质质感

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 白板面板 | 微黄背景#f7f3e3，A4比例画布，Canvas绘制层，笔触黑色2px |
| 主页面 | 甘特图面板 | 白色背景，周级别时间轴，彩色条块（按优先级），进度渐变填充 |
| 主页面 | 分组面板 | 背景#fafafa，宽220px，三角箭头图标，展开/折叠过渡0.2s |
| 主页面 | 分割线 | 宽6px，颜色#ddd，cursor col-resize，拖拽高亮#a0784c |
| 主页面 | 导出按钮 | 圆形直径36px，悬停#e8e8e8，下拉菜单PNG/PDF |
| 主页面 | 甘特图条块 | 悬停透明度0.9→1，上升4px，过渡0.25s缓出 |

### 4.3 响应式设计
- 桌面优先设计（1366px以上）：双栏并排布局
- 中等屏幕（1024px-1366px）：双栏自适应
- 小屏幕（1024px以下）：上下堆叠，白板在上、甘特图在下，间距16px

### 4.4 性能要求
- 从停止书写到甘特图生成 ≤ 1.5秒
- 甘特图条块超过200个时帧率 ≥ 30fps
