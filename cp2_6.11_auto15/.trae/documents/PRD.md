
## 1. 产品概述
一款运行在浏览器中的策略塔防游戏，解决传统塔防游戏路径单一、防御塔类型过少的痛点。通过自适应寻路AI迫使玩家构建多路径防御策略，提供五种特色防御塔与多样化敌人类型，带来深度策略体验。
- 目标用户：喜欢策略类游戏玩家，追求深度塔防游戏爱好者
- 产品价值：通过实时自适应寻路+多塔种策略深度+精美暗色科幻视觉体验

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 玩家 | 无需注册，直接游玩 | 建造/升级/出售防御塔，开始波次，查看统计 |

### 2.2 功能模块
1. **游戏主场景：地图网格、防御塔建造区域、粒子星空背景
2. **防御塔系统：5种塔（箭塔、炮塔、冰塔、毒塔、电塔），每塔3级升级
3. **敌人系统：5种敌人（普通、快速、重甲、飞行、首领），自适应A*寻路
4. **波次系统：波次生成、15秒准备时间、波次总结简报
5. **UI面板：实时统计面板、防御塔选择面板、升级/出售选项
6. **特效系统：建造/升级粒子动画、敌人受击特效、塔生长动画

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 游戏主界面 | 游戏地图区域 | 网格化放置防御塔，敌人行进与敌人自适应寻路，防御塔攻击敌人 |
| 游戏主界面 | 底部状态栏 | 显示金币数量、当前波次、玩家生命值 |
| 游戏主界面 | 右侧塔选择面板 | 展示5种防御塔图标，显示解锁状态/价格/可用状态 |
| 游戏主界面 | 右侧升级面板 | 选中塔后显示升级选项、出售选项、塔属性详情 |
| 游戏主界面 | 实时统计面板 | 显示击杀数、防御塔数量、各塔升级状态 |
| 游戏主界面 | 波次简报弹窗 | 波次结束时弹出总结本波统计、击杀奖励、波次奖励金币 |

## 3. 核心流程
玩家进入游戏 → 初始金币建造防御塔 → 等待15秒准备 → 开始波次 → 敌人自适应寻路 → 防御塔攻击 → 波次结束 → 结算奖励 → 下一波准备 → 循环直至玩家生命值归零或通关。

```mermaid
flowchart TD
    A["游戏开始"] --> B["15秒准备期"]
    B --> C["玩家建造/升级塔"]
    C --> D{"开始波次"}
    D --> E["敌人生成"]
    E --> F["A*寻路"]
    F --> G["防御塔攻击"]
    G --> H{"全部清除?"]
    H -->|否| F
    H -->|是| I["波次结算"]
    I --> J["生命值>0?"]
    J -->|是| B
    J -->|否| K["游戏结束"]
```

## 4. 用户界面设计
### 4.1 设计风格
- 主色调：深蓝 (#0a1628) 和亮青色 (#00f0ff)
- 辅助色：红色警示 (#ff3366)、金色奖励 (#ffd700)
- 按钮风格：圆角(8px)、半透明毛玻璃 (backdrop-blur)、悬浮亮青色边框发光
- 字体：无衬线现代字体，标题18px，正文14px
- 布局风格：卡片式半透明面板，深蓝基底，亮青色高光边框
- 图标风格：几何抽象风格，状态指示环(未锁定-灰色、可用-亮青色、升级中-金色脉冲)

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 游戏主界面 | 地图区域 | 深蓝渐变网格，敌人行进路径动态高亮，塔放置区网格线 |
| 游戏主界面 | 底部状态栏 | 半透明毛玻璃卡片，金币/波次/生命值图标+数值，生命值渐变条 |
| 游戏主界面 | 右侧塔面板 | 5个塔卡片，塔图标+名称+价格，选中高亮发光 |
| 游戏主界面 | 升级面板 | 塔详情：等级条，升级/出售按钮，属性数值列表 |
| 游戏主界面 | 统计面板 | 击杀/塔数/波次进度条 |
| 游戏主界面 | 波次简报 | 居中半透明弹窗，击杀数+金币奖励+继续按钮 |
| 游戏主界面 | 粒子星空背景 | 缓慢流动粒子，深蓝星空渐变 |

### 4.3 响应式
- Desktop-first，1920x1080与1440x900双分辨率适配
- 所有UI使用百分比和vw/vh相对单位
- Canvas自适应屏幕尺寸，UI面板固定位置但内边距自适应

### 4.4 Canvas渲染性能
- 30个敌人同时存在时稳定60fps
- 对象池复用粒子特效
- requestAnimationFrame驱动
- 逻辑更新与渲染分离
