# 赛博朋克黑客攻防战 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
一款赛博朋克风格的实时网络拓扑策略游戏，玩家扮演黑客在节点网络图中入侵服务器，与AI对抗，最终夺取核心数据。

### 1.2 目标用户
喜欢策略类、赛博朋克题材的游戏玩家。

### 1.3 核心价值
- 解决传统回合制策略游戏缺乏实时操作感的问题
- 提供直观的网络拓扑可视化，让玩家清晰理解攻防局势
- 将黑客攻防抽象为可操作的节点占领策略玩法

---

## 2. 功能需求

### 2.1 地图与节点系统
- **图结构生成**: 动态生成至少12个节点、20条边的无向图
- **节点形态**: 六边形3D模型，蓝色为玩家节点，红色为AI节点，青色为中立节点
- **边连接**: 节点间的发光连线，表示网络通路
- **交互能力**:
  - 点击相邻节点发起移动
  - 点击相邻敌方/中立节点发起攻击
  - 攻击时节点周围爆发脉冲电光动画
  - 攻占后节点变色并扩散涟漪效果
- **特殊节点**: 地图最中心的核心数据节点，需连续攻击两回合才能夺取

### 2.2 AI系统
- 实时接管未被占领的节点
- 主动追击玩家节点
- 可使用脚本进行反制

### 2.3 脚本系统
三种可使用的脚本，每次消耗能量值:

| 脚本名称 | 能量消耗 | 效果 |
|---------|---------|------|
| 破译脚本 (Crack) | 3点 | 加速攻占节点 |
| 防火墙脚本 (Firewall) | 2点 | 加固已占领节点 |
| 隐形脚本 (Stealth) | 4点 | 暂时隐藏移动轨迹 |

- 初始能量值: 10点
- 每回合自动恢复: 3点
- 使用时右下角显示半透明终端弹窗，绿色等宽字体逐字显示脚本名称，随后0.3秒扫描线动画

### 2.4 核心数据夺取与胜利条件
- 夺取中心核心数据节点后，进入5秒倒计时最终阶段
- 玩家需将数据文件拖拽到自己的起始节点才算胜利
- 拖拽过程中数据文件以脉冲光球形式跟随鼠标
- AI集合所有剩余节点追踪玩家，追踪路径用红色虚线实时显示

### 2.5 UI界面

#### 左侧面板
- 玩家状态展示: 能量条、当前脚本三个槽位
- AI行动日志: 滚动列表，每个条目带时间戳和技能名称

#### 右侧3D视图
- 动态图网络可视化
- 鼠标拖拽平移
- 滚轮缩放
- 数据传输时连接线闪烁
- 玩家移动时路径节点依次高亮

## 3. 非功能需求

### 3.1 性能要求
- 帧率稳定 ≥ 45 FPS
- 图渲染单帧耗时 ≤ 30ms

### 3.2 视觉风格
- **主题**: 深色赛博朋克
- **主背景色**: #0b0c10
- **辅助色**: #66fcf1
- **节点默认色**: #45a29e
- 整体科技感、霓虹发光效果

### 3.3 技术栈
- TypeScript
- Three.js (3D节点和连线渲染)
- Phaser3 (状态管理和回合逻辑)
- Vite (构建工具)

## 4. 文件结构

```
├── package.json
├── vite.config.js
├── tsconfig.json
├── index.html
└── src/
    ├── network/
    │   ├── GraphManager.ts      # 图结构生成、坐标计算、路径查找

    └── game/
        ├── GameState.ts         # 核心状态机（回合计数、占领、能量、冷却）

        ├── AIController.ts      # AI逻辑
        └── UIManager.ts         # HTML/CSS UI界面
```
