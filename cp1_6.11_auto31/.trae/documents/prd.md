## 1. 产品概述

CodeSandbox Mini 是一个面向前端开发者与学习者的在线代码编辑器，解决快速测试 HTML/CSS/JavaScript 代码片段而无需打开完整编辑器或新建文件的问题。
- 目标用户：前端开发者、编程学习者、需要快速原型验证的工程师
- 核心价值：即写即见、零配置、轻量便携的代码实验环境

## 2. 核心功能

### 2.1 用户角色
| 角色 | 使用方式 | 核心权限 |
|------|----------|----------|
| 所有用户 | 直接访问 | 编辑、预览、导出、历史恢复 |

### 2.2 功能模块
1. **编辑器页面**：分栏布局代码编辑区 + 实时预览区，三标签页切换，工具栏操作

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 编辑器页面 | 代码编辑区 | 三个标签页（HTML/CSS/JS），支持语法高亮、自动补全、括号匹配、行号显示 |
| 编辑器页面 | 实时预览区 | iframe 实时渲染，防抖 300ms 自动刷新，JavaScript 错误捕获显示 |
| 编辑器页面 | 工具栏 | 运行、重置、导出、全屏预览按钮，历史记录下拉菜单 |
| 编辑器页面 | 错误提示栏 | 红色背景显示 JS 错误信息及行号，点击可跳转对应行 |
| 编辑器页面 | 可拖拽分隔条 | 调整编辑区与预览区宽度比例，弹性缓动效果 |

## 3. 核心流程

用户在左侧编辑区编写 HTML/CSS/JS 代码，输入后 300ms 防抖自动刷新右侧预览区。用户可通过工具栏手动运行、重置代码、导出为 HTML 文件、切换全屏预览。JS 运行错误在底部错误栏展示。最近 5 次编辑自动保存到 localStorage，可通过历史下拉恢复。

```mermaid

    "JS 执行报错?" --> "否: 正常渲染"
    "用户点击运行" --> "拼接 HTML+CSS+JS" 
    "用户点击重置" --> "清空所有代码区"
    "用户点击导出" --> "下载合并 HTML 文件"
    "用户点击全屏" --> "预览区全屏展示"
    "代码变更" --> "保存到 localStorage (最多5条)"
    "用户点击历史记录" --> "恢复选中历史代码"
```

## 4. 用户界面设计

### 4.1 设计风格
- 主背景色：深灰 #1e1e1e，代码区更深 #121212
- 按钮风格：扁平化，悬停时背景 rgba(255,255,255,0.08)，微上升 2px，点击缩放 0.95
- 字体：等宽字体用于代码编辑区，UI 字体选用 JetBrains Mono / Fira Code
- 布局风格：左右分栏（编辑区 + 预览区），顶部工具栏，底部错误栏
- 标签页颜色：HTML 橙色 #e44d26、CSS 蓝色 #264de4、JavaScript 黄色 #f7df1e

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 编辑器页面 | 工具栏 | 深灰背景，图标按钮，历史下拉菜单（250px宽，从右滑入） |
| 编辑器页面 | 代码编辑区 | CodeMirror 暗色主题，三标签页（带颜色下划线动画），行号 |
| 编辑器页面 | 可拖拽分隔条 | 窄条，悬停 col-resize 光标，拖拽弹性缓动 |
| 编辑器页面 | 实时预览区 | iframe，右上角 loading 旋转图标（18px，#6c9ca6，1.5s/周） |
| 编辑器页面 | 错误提示栏 | 红色背景，白色文字，显示错误信息和行号，可点击跳转 |

### 4.3 响应式设计
- 桌面优先设计，屏幕宽度 < 768px 时编辑区和预览区改为上下叠放
- 移动端标签页改为滚动式水平排列
- 各占约 50% 高度

### 4.4 动效细节
- 标签页切换：选中标签下方 2px 下划线，0.3s 平滑过渡
- 工具栏按钮悬停：背景 rgba(255,255,255,0.08)，上升 2px，过渡 0.2s
- 工具栏按钮点击：缩放 0.95 后恢复
- 分隔条拖拽：0.1s 弹性缓动
- 历史菜单：从右向左滑入展开
- Loading 图标：旋转动画，1.5s 一周匀速
