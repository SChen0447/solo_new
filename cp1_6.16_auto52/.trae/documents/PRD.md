# 回合制RPG战斗系统设计工具 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
面向独立游戏策划的浏览器端回合制RPG战斗系统设计与验证工具，解决手动平衡角色属性、技能效果与敌方AI逻辑时，难以快速找到最佳数值且缺乏可视化反馈的问题。

### 1.2 目标用户
- 独立游戏策划
- RPG数值设计师
- 游戏开发者

### 1.3 核心价值
- 快速配置英雄和敌方阵容
- 即时战斗模拟与可视化反馈
- 多回合数据统计与平衡验证
- 数据导出支持深度分析

---

## 2. 功能需求

### 2.1 角色阵容与技能编辑模块

| 功能点 | 描述 | 约束条件 |
|--------|------|----------|
| 英雄创建 | 左侧面板创建最多4个英雄角色 | 1 ≤ 数量 ≤ 4 |
| 基础属性配置 | 配置名称、职业、生命值、攻击力、防御力、速度 | HP: 100-500, ATK: 10-50, DEF: 5-30, SPD: 1-10 |
| 职业类型 | 战士/法师/刺客/牧师 | 4种预设职业 |
| 技能预设 | 每个角色预设3个技能 | 固定3个技能槽位 |
| 技能编辑 | 编辑名称、类型、消耗、冷却、效果数值 | MP消耗: 0-30, 冷却: 0-3回合 |
| 技能类型 | 物理/魔法/治疗/增益/减益 | 5种技能类型 |
| 即时存储 | 技能编辑完成后立即存储到角色数据中 | 无确认按钮，即时生效 |

### 2.2 敌方波次自动生成模块

| 功能点 | 描述 | 约束条件 |
|--------|------|----------|
| 敌人模板选择 | 选择预定义敌人模板组成小队 | 哥布林/骷髅/暗精灵/石头人/Boss龙 |
| 敌方编队 | 组成1-5个敌人的小队 | 1 ≤ 数量 ≤ 5 |
| 属性微调 | 微调每个敌人的生命、攻击、防御、速度 | 基于模板默认值调整 |
| 自动AI生成 | 自动生成简单行为AI | 优先攻击HP最低英雄、30%技能概率、治疗嘲讽 |

### 2.3 战斗模拟与回合日志模块

| 功能点 | 描述 | 约束条件 |
|--------|------|----------|
| 自动战斗推进 | 回合制自动推进 | 每回合0.3秒间隔 |
| 速度排序 | 所有参与方按速度排序行动 | 速度高者先行动 |
| 时间戳日志 | 每个行动生成时间戳日志 | 精确到回合+毫秒 |
| 日志颜色区分 | 攻击红色、治疗绿色、增益蓝色、减益橙色 | #FF4444/#44AA44/#4488FF/#FF8800 |
| 自动滚动 | 日志区域自动滚动到底部 | 新日志追加后触发 |
| 生命值条显示 | 实时显示双方生命值条 | 英雄蓝渐变，敌人红渐变 |
| 低血量特效 | HP<20%时血条闪烁并变深红色 | 视觉警示效果 |
| 技能闪光特效 | 技能释放时放大1.1倍再归位 | 持续0.15秒 |

### 2.4 战斗结果统计模块

| 功能点 | 描述 | 约束条件 |
|--------|------|----------|
| 战斗结束判定 | 一方全部阵亡或达到50回合 | 最大50回合限制 |
| 结果面板弹出 | 展示胜方/败方和统计数据 | 模态弹窗形式 |
| 胜负展示 | 奖杯图标(金色)+结果文字 | 0.5秒淡入动画 |
| 英雄统计 | 总伤害/治疗/承伤/击杀数 | 每位英雄独立统计 |
| 敌方统计 | 每个单位存活回合数 | 敌方全体统计 |
| 总回合数显示 | 展示战斗持续回合数 | - |

### 2.5 多回合模拟与数据导出模块

| 功能点 | 描述 | 约束条件 |
|--------|------|----------|
| 模拟次数设置 | 1-100次，默认1次 | 随机种子不同 |
| 柱状图统计 | 展示英雄平均伤害和平均治疗 | 使用recharts |
| CSV导出 | 将每次模拟详细日志导出为CSV | 一键下载 |

## 3. 界面与交互需求

### 3.1 视觉规范

| 元素 | 颜色值 | 说明 |
|------|--------|------|
| 背景色 | #1E1E2E | 深色主题底色 |
| 卡片/面板 | #2B2B3D | 次深色容器 |
| 主文字色 | #E0E0E0 | 主内容文字 |
| 英雄头像渐变 | #6C63FF → #A29BFE | 蓝紫圆形渐变 |
| 敌人头像渐变 | #FF6B6B → #FF8E53 | 红橙圆形渐变 |
| 分隔线 | #3A3A4C | 1px浅灰分隔 |

### 3.2 布局规范

| 区域 | 宽度 | 说明 |
|------|------|------|
| 左侧编辑面板 | 320px | 可折叠，0.2秒滑动动画 |
| 中央战斗区 | 自适应 | 占满剩余宽度 |
| 右侧日志面板 | 300px | 可滚动 |

### 3.3 响应式要求

- **≥768px**: 三栏布局并排显示
- **<768px**: 左右面板变为浮动层，通过顶部汉堡菜单切换

---

## 4. 性能需求

| 指标 | 要求 |
|------|------|
| 单回合日志+UI更新 | ≤2ms |
| 50回合模拟显示 | ≤1秒 |
| 100次模拟总计算 | ≤1秒，不阻塞主线程 |

---

## 5. 技术栈要求

- **语言**: TypeScript
- **框架**: React 18
- **构建工具**: Vite
- **图表库**: recharts
- **其他**: uuid
