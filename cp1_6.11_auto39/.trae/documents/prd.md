## 1. 产品概述

会议管家（MeetingKeeper）是一款面向组织和个人的全栈会议管理应用，旨在解决跨部门会议中信息分散、后续跟进困难的问题。
- 核心价值：将会议创建、实时记录、纪要生成、待办追踪整合为一体化工作流，让每一次会议都有据可查、有果可追
- 目标用户：需要频繁组织或参与跨部门会议的团队管理者、项目经理、行政人员

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 系统预设账号登录 | 创建会议、查看会议、实时记录、管理待办 |
| 主持人 | 创建会议时自动成为 | 以上权限 + 控制会议流程、标记议程完成/跳过 |

### 2.2 功能模块

1. **登录页**：用户身份认证，登录后进入主页
2. **主页**：历史会议列表，按最近创建排序，展示进度条和会议概要
3. **会议创建页**：填写会议信息、添加参与人、编排议程条目
4. **实时记录页**：主持人进入实时记录模式，逐条议程记录，自动保存
5. **会议详情/纪要页**：查看完整纪要、管理待办事项、导出Markdown

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 登录页 | 登录表单 | 输入用户名密码，调用登录API，验证后存储token跳转主页 |
| 主页 | 会议卡片列表 | 显示所有会议，每张卡片含主题、日期、待办完成率进度条，点击进入详情 |
| 主页 | 创建会议入口 | 右上角按钮，点击进入会议创建页 |
| 会议创建页 | 基本信息表单 | 会议主题（非空校验）、日期时间（不能为过去）、参与人搜索添加（防抖300ms） |
| 会议创建页 | 议程编排 | 动态添加/删除议程条目，每条含标题和预估时长（正整数校验），fadeIn/fadeOut动画 |
| 实时记录页 | 议程列表 | 按议程顺序展示，每条下方有富文本编辑器（加粗、列表、重要标记） |
| 实时记录页 | 自动保存 | 每5秒自动保存，状态指示器显示"已保存/保存中/未保存" |
| 实时记录页 | 议程状态 | 完成（自动计算实际耗时）或跳过，完成后显示耗时 |
| 会议详情页 | 纪要报告 | 结构化展示会议基本信息、议程记录、讨论摘要 |
| 会议详情页 | 待办事项卡片网格 | 显示事项描述、责任人、截止日期，过期未完成闪烁红色边框 |
| 会议详情页 | 待办事项管理 | 手动添加、编辑、删除待办，指派责任人 |
| 会议详情页 | Markdown导出 | 一键导出结构化会议纪要为.md文件 |

## 3. 核心流程

用户登录后，在主页查看历史会议列表。点击"创建会议"进入表单页，填写主题、时间、参与人并编排议程条目。会议开始后，主持人进入"实时记录"模式，逐条议程进行记录，系统每5秒自动保存。每个议程可标记"完成"或"跳过"，完成时自动计算实际耗时。会议结束后，系统自动生成结构化纪要报告并提取待办事项。用户可在报告页面管理待办事项、指派责任人，并一键导出Markdown格式纪要。

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：蓝灰色系（背景 #2c3e50，强调色 #3498db）
- 辅助色：白色卡片背景、#e74c3c（警告/过期）、#27ae60（完成/成功）、#95a5a6（中性灰）
- 按钮风格：圆角按钮（border-radius: 8px），主操作按钮使用强调色填充，次要操作使用边框样式
- 字体：系统字体栈（-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif），标题18-24px，正文14-16px
- 布局风格：顶部导航栏 + 内容区域居中（最大宽度1200px），卡片式布局
- 图标：使用 lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 登录页 | 登录表单 | 居中卡片，蓝色渐变背景，输入框带focus边框动画，登录按钮hover上浮 |
| 主页 | 会议卡片列表 | 网格布局卡片，圆角12px，轻微阴影，hover上浮动画translateY(-2px) 0.2s ease |
| 主页 | 进度条 | 卡片内嵌进度条，蓝色填充，宽度动画过渡 |
| 会议创建页 | 表单 | 分区卡片布局，输入框实时校验红色边框提示，参与人搜索下拉列表 |
| 会议创建页 | 议程列表 | 可增删的列表，fadeIn/fadeOut动画，拖拽排序视觉提示 |
| 实时记录页 | 议程卡片 | 当前议程高亮，富文本工具栏，自动保存状态指示器（右上角胶囊标签） |
| 实时记录页 | 计时器 | 当前议程实时计时显示 |
| 会议详情页 | 纪要报告 | 结构化卡片排列，议程标题+内容+耗时，待办提取区域 |
| 会议详情页 | 待办卡片网格 | 多列网格布局，卡片含描述/责任人/截止日期，过期卡片红色边框闪烁动画 |
| 会议详情页 | 导出按钮 | 右上角操作栏，下载图标按钮 |

### 4.3 响应式设计

- 桌面优先设计，最大宽度1200px居中
- 平板（≤768px）：议程输入框独占一行，待办卡片双列布局
- 手机（≤480px）：待办卡片单列布局，导航栏折叠为汉堡菜单，表单全宽

### 4.4 骨架屏与加载状态

- 页面初始加载时使用骨架屏占位（灰色脉冲动画 animate-pulse）
- 骨架屏形状匹配实际内容区域（卡片形、文字条形）
- API请求期间显示加载指示器
