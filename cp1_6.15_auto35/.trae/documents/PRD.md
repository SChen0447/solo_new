## 1. 产品概述

声波战斗技能映射与实时反馈原型是一款面向独立游戏开发者的交互式测试工具，用于验证"声音可视化战斗"机制。用户可通过麦克风或键盘输入触发不同频率和节奏的声音，系统实时解析声音特征并映射为战斗技能，配合粒子特效和数据反馈完成完整的游戏体验验证。

- 目标用户：独立游戏开发者、游戏设计师、声音交互研究者
- 产品价值：为声音驱动的游戏机制提供快速原型验证，降低设计迭代成本

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 开发者用户 | 无需注册，直接使用 | 完整功能访问、参数调试、技能映射测试 |

### 2.2 功能模块
1. **战斗场景主界面**：Canvas 渲染的 2D 战斗区域、视差背景、地面线效果
2. **声音输入系统**：麦克风实时采集、FFT 频率分析、音量与节奏检测
3. **技能映射引擎**：5 种战斗技能的声学条件匹配、冷却管理、连锁判定
4. **粒子特效系统**：技能释放时的粒子生成、动画、拖尾效果、屏幕震动
5. **实时数据面板**：音高条形图、音量圆环、BPM 数值、技能命中率环形进度
6. **模拟键盘输入区**：5 个虚拟按钮支持无麦克风环境测试

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主界面 | 战斗场景 Canvas | 800×500px 黑色背景，底部发光地面线，粒子特效叠加渲染 |
| 主界面 | 录音控制按钮 | 圆形 48px 直径，灰色默认状态，录音时红色脉动（1Hz），点击切换麦克风 |
| 主界面 | 虚拟技能按钮区 | 5 个 80×60px 彩色按钮，对应 5 种技能，悬停放大 1.05 倍，0.15s 过渡 |
| 主界面 | 数据面板 | 右上角 220×300px 半透明面板，60fps 实时更新音高、音量、节奏、命中率 |

## 3. 核心流程

用户启动应用 → 选择输入方式（麦克风/虚拟按钮） → 发出声音或点击按钮 → SoundEngine 解析频率/振幅/节奏 → GameLoop 匹配技能映射表 → 触发粒子特效与屏幕震动 → 更新命中率与技能冷却 → 实时数据面板刷新显示

```mermaid

    C --> E["技能状态更新（冷却/命中率）"]
    D --> F["Canvas 战斗场景渲染"]
    E --> G["UIPanel 数据面板"]
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深空蓝 `#0a0a1a`，深蓝渐变背景
- 技能配色：音波冲击 `#00d4ff`（青）、护盾共鸣 `#ffd700`（金）、音刃斩 `#ff4444`（红）、震荡波 `#aa00ff`（紫）、回声陷阱 `#00ff88`（绿）
- 字体：monospace 等宽字体，全白色文本
- 按钮风格：圆角 8px，悬停放大过渡 0.15s
- 布局：战斗场景居中 800×500px，数据面板右上角固定，按钮区下方居中

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 主界面 | 战斗场景 | 黑色背景 #000011，浅蓝半透明地面线带光晕，视差网格背景 #1a1a3a 间隔 40px 透明度 0.3 |
| 主界面 | 数据面板 | rgba(0,0,0,0.7) 背景，圆角 12px，宽 220px，外边距 10px |
| 主界面 | 录音按钮 | 圆形 48px，默认 #ccc，录音时 #ff3333 带 1Hz 脉动光圈 |
| 主界面 | 虚拟按钮 | 5 个彩色按钮，间距 12px，居中对齐，冷却时灰色显示倒计时 |

### 4.3 响应式设计
- Desktop-first 设计，优先适配 1280px 及以上分辨率
- 战斗场景保持固定 800×500px 尺寸
- 数据面板绝对定位，不受布局影响
- 移动端自动缩小按钮尺寸，保持可用性

### 4.4 性能约束
- 同时存在粒子数 ≤ 100
- 帧率维持 60fps
- 声音分析延迟 < 50ms
- UI 更新无卡顿
