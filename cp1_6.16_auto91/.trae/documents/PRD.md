## 1. 产品概述

AI文案辩论评估平台，帮助营销和内容创作团队通过多轮辩论与实时投票机制，对AI生成的备选文案进行质量评估和排序，高效达成团队共识。

- 核心价值：解决团队面对大量AI生成文案时难以高效筛选和达成共识的痛点
- 目标用户：市场营销团队、内容创作者、品牌运营人员

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 生成文案、参与辩论、投票、查看历史记录 |

### 2.2 功能模块
1. **文案生成面板**：输入产品信息，生成4条不同风格的备选文案
2. **多轮辩论区**：两轮辩论模式，第一轮评论，第二轮点赞
3. **投票与结果展示**：倒计时投票、柱状图排名展示
4. **历史记录侧边栏**：历史辩论记录管理和回放

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 文案生成表单 | 输入产品名称、目标受众、核心卖点，触发生成 |
| 主页面 | 文案卡片展示 | 4条文案以棋盘式动画入场，显示风格标签 |
| 主页面 | 第一轮辩论区 | 每条文案下方评论输入框，支持200字限制和实时统计 |
| 主页面 | 第二轮辩论区 | 最热两条文案的匿名评论，带动物头像和点赞功能 |
| 主页面 | 投票区 | 1分钟倒计时，单选投票，动画反馈 |
| 主页面 | 结果展示 | 横向柱状图排名，显示得票数和百分比 |
| 侧边栏 | 历史记录列表 | 已完成辩论的摘要列表，支持展开/收起 |
| 侧边栏 | 历史回放 | 按时间线重播辩论过程，0.5倍速 |

## 3. 核心流程

用户输入产品信息 → 点击生成按钮 → 后端返回4条不同风格文案 → 棋盘式动画展示 → 第一轮辩论（用户对每条文案评论）→ 第二轮辩论（最热2条文案匿名点赞）→ 1分钟投票倒计时 → 用户投票 → 柱状图展示排名结果 → 辩论记录自动保存 → 可随时查看历史记录和回放

## 4. 用户界面设计

### 4.1 设计风格
- **主题**：暗色主题，背景 #1E1E1E，主文字 #E0E0E0
- **强调色**：#FF5722（主强调）、#4CAF50（成功/选中）、#F44336（警示/倒计时）
- **卡片样式**：12px圆角矩形，内嵌投影 0 4px 12px rgba(0,0,0,0.3)
- **字体**：主体14px，风格标签10px
- **交互动效**：所有可交互元素hover时0.2秒透明度变化（0.8）+ 轻微上浮（translateY(-2px)）

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 文案生成表单 | 三个输入框（产品名称、目标受众、核心卖点），橙色生成按钮 |
| 主页面 | 文案卡片 | 4张卡片2x2布局，左上角橙色风格标签，棋盘式交错入场动画 |
| 主页面 | 第一轮辩论区 | 卡片下方输入框，灰色字数统计，提交后右下角气泡评论 |
| 主页面 | 第二轮辩论区 | 高亮2条"待辩论文案"，匿名评论带随机动物头像，大拇指点赞按钮 |
| 主页面 | 投票区 | 圆形单选按钮，选中绿色填充+放大回弹动画，红色倒计时每秒跳动 |
| 主页面 | 结果展示 | 横向柱状图，金色到橙色渐变，从底部向上填充动画 |
| 侧边栏 | 历史记录 | 左侧滑出动画，列表项显示时间、产品、前三摘要 |

### 4.3 响应式设计
- **桌面端（>1024px）**：三栏布局 - 左侧历史栏250px，中间内容区自适应，右侧投票面板320px
- **平板端（768-1024px）**：双栏布局 - 侧边栏折叠为顶部导航，中间内容区占满
- **手机端（<768px）**：单列全宽堆叠，侧边栏变为底部抽屉

### 4.4 动效规范
- 文案卡片入场：依次从下方滑入，间隔0.1秒，ease-out缓动
- 评论气泡：头像右侧横向展开0.3秒
- 投票按钮：选中放大-回弹0.2秒
- 柱状图：从底部向上拉伸0.5秒，帧率≥30fps
- 侧边栏：从左侧滑出0.3秒
- 倒计时：每秒跳动式减少
- 点赞：点赞后变为实心并微动
